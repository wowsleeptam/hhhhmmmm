1) Существуют два тип отношения между объектами - компазиция, агрегация. Часть компазиции
может быть частью только у одного объекта (класса). Эта часть управляется объектом (класом)
Часть не знает о сущестовании объекта (класса)
При агрегации часть является частью класса, часть может пренадлежать болле чем одному классу
часть существует, не управляемая классом, также часть не знает о сущестовании класса
UPD: Также существует ассоциация, в которой объекты могут быть связаны с другими классами,
помипо исходного и могут перемещать от объектов одного класса к другому 

2) Есть 3 типа наследования: public, private, protected. Public делает открывает доступ для
всех членов родительского класса. private - дочерние классы не могу обратиться к данным родительского класса
protected - открывает доступ к членкам класса дочерним классам 

3) Сигнатура виртуального метода дочернего класса должна полностью соответствовать сигнатуре 
виртуального метода родительского класса. Если у дочернего метода будет другой тип параметров, 
нежели у родительского, то вызываться этот метод не будет.
UPD: При множестенном наследовании (если дочерний класс используется в 2-ух разных родительских классах)
у каждного родительского класса есть один и тот же дочерний класс. При использовании виртуальный базовый класс
мы даем "контроль нескольким родительским классам на использование одной копии дочернего класса"

4) Вертуальные фукнция решают проблему перегрузки фукнций. Ключевое слово virtual заставляет выполнить
"наиболее дочернюю" фукнцию при вызове, что позволяет решить большество перегрузок при наследовании
